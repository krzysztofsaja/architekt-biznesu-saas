# Scroll-to-Give

## Opis Projektu

**One-Liner:** Aplikacja, która zamienia czas spędzony na scrollowaniu w social media na "wirtualną darowiznę" dla wybranych celów charytatywnych – budując świadomość nawyków bez realnych kosztów.

## Model Biznesowy

### Wersja MVP (Symulacja)

- **Social Credits:** Aplikacja oblicza i wyświetla "wartość" czasu spędzonego na scrollowaniu (np. 1 minuta = 0,10 zł)
- **Tip Jar:** Opcjonalna dobrowolna dotacja na rozwój projektu (Buy Me a Coffee / PayPal)
- **Gamifikacja:** Streaki, odznaki, tablice wyników

### Wersja Rozszerzona (przyszłość)

- Integracja z prawdziwymi organizacjami charytat.
- Mikropłatności (Stripe)
- Więcej statystyk i personalizacji

## Grupa Docelowa

- Osoby, które chcą ograniczyć czas w social media
- Ludzie z wyrzutami sumienia po godzinach scrollowania
- Osoby, które próbowały innych metod (bloczek, aplikacje typu Forest) i nie zadziałały

## Kluczowe Funkcje MVP

1. **Dashboard:** Pokazuje łączną "wirtualną darowiznę" do tej pory
2. **Manual Input:** Użytkownik wprowadza czas (np. "Dziś 2h na TikToku")
3. **Wybór Celu:** Lista predefiniowanych celów (zwierzęta, edukacja, zdrowie)
4. **Kalendarz:** Historia wpisów, wizualizacja postępów
5. **Streaky:** Licznik dni z rzędu z ograniczonym scrollowaniem

## Stack Technologiczny (Dla Początkującego)

- **Frontend:** HTML + CSS + Vanilla JavaScript (najprostsze do nauki)
- **Backend:** Firebase (darmowy tier, bez serwera do konfiguracji)
- **Hosting:** GitHub Pages (darmowe)

## Harmonogram (5-10h/tydz.)

| Etap | Opis                                  | Szacowany czas |
| ---- | ------------------------------------- | -------------- |
| 1    | Setup środowiska (HTML/JS + Firebase) | 2-3h           |
| 2    | Struktura dashboard + UI              | 5-8h           |
| 3    | Logika obliczania "Social Credits"    | 3-5h           |
| 4    | Firebase integration (zapis danych)   | 5-8h           |
| 5    | Kalendarz + wizualizacja              | 5-8h           |
| 6    | Testy + deploy                        | 2-3h           |

**Łącznie:** ~22-35h (4-7 tygodni przy 5h/tydz.)

## Ryzyka i Ograniczenia

- **Ręczne wprowadzanie:** Użytkownik może "oszukać" – ale to też buduje świadomość
- **Motywacja:** Bez realnych konsekwencji finansowych – działanie zależy od użytkownika
- **Brak automatyzacji:** W przyszłej wersji można dodać rozszerzenie Chrome
